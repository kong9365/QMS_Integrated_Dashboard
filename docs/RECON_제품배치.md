## 3) 체인 분포 — 전파(propagation) 필요성 판정

`build_ctx`(부모-자식 그래프)로 **양쪽 모두 룩업 가능한 부모-자식 쌍 5,876건**에서 품목/lot 채움율을 비교.

**전체 요약**

| 필드 | 부모 채움 | 자식 채움 | 부모有·자식無(전파 후보) |

| `품목코드` | 320 / 5876 (5.4%) | **0 / 5876 (0.0%)** | **320 쌍** |

### §5 제품·배치 품질 — 서브탭 A (APQR)
```diff
- **품목 식별**: `시험정보목록[].품목코드`(itemCd/matnr/zzinmat, extract_test_item_code) + `품목명`(itemNmReport).
+ **품목 식별**: 파싱 `품목코드`/`품목명` 보유는 **OOS(100%)·일탈(82.1%) 둘뿐**.
+   조사(investigation)는 품목 컬럼 부재 → 직접 필터 불가, 부모(OOS/일탈) 품목 **체인 상속**으로만 귀속.
- **매트릭스 행(집계 가능)**: OOS 발생 / 일탈(등급가중) / 조사 건수 / CAPA 이행률(체인 상속) / 재발 건수.
-   모두 `품목코드`로 필터 후 `연도`별 건수기여도 합.
+ **매트릭스 행**: OOS·일탈은 `품목코드` 직접 필터; **조사·CAPA·Action 은 품목코드 부재(자식 0.0%) → 부모 품목 전파 필요**(미구현).
+   연도 축은 `연도`(전 프로젝트 100%); 조사는 `발견일시` 부재라 `등록일`/`연도` 사용.
+   '재발 건수' 행은 §6 재발 정정(아래) 선결 필요.
```

### §5 제품·배치 품질 — 서브탭 B (lot 디스포지션)
```diff
- **수집**: 해당 lot의 직접보유 이벤트(OOS/일탈/조사 — `제조번호` 매칭) + 체인 자식(CAPA/Action).
+ **수집**: `제조번호` 실채움은 **OOS 만(100%)**. 일탈은 `제조번호` 컬럼 존재하나 **0%(빈값)**, 조사는 컬럼 부재.
+   → lot 직접 매칭은 사실상 **OOS 전용**; 일탈·조사는 lot 키로 못 잡고 **체인(상위번호)**으로만 연결.
  **판정**: 모든 관련 이벤트의 `최종 종결 여부(체인)` True→PASS, 하나라도 미종결→HOLD.   # (필드 유효 — OOS/일탈/조사 100% 보유)
+   보조: OOS `기준 일탈 최종 결과`(적합 1300 / 부적합 35 / 빈값 16) 병기 가능.
```

### §6 알림·모니터링 — 재발 발생 룰  &  §1 이상신호 '재발'
```diff
- | 재발 발생 | `재발여부 == "예"` |
+ | 재발 발생 | ⚠️ 실데이터에 `'예'` **0건** → 이 조건은 항상 0. 실값 분포: `''`=634·`'2'`=21·`'6'`=13·`'아니오'`=1(deviation 단독).
+              현 코드 정의(notna·비어있지않음·`'아니요'` 제외)와도 철자 불일치(`'아니오'`≠`'아니요'`).
+              → '재발 신호' 정의 재합의 필요(숫자코드 2·6 의 의미 확인 포함). deviationoutsourcing 은 0행이라 deviation 단독. |
```
> §1 종합현황 '이상신호 — 재발'(`재발여부 == "예"`, deviation/deviationoutsourcing 최근 90일) 동일 정정 대상.
> deviationoutsourcing 은 stub 0행이므로 "deviation/deviationoutsourcing" → **deviation** 로 축소 표기 권장.
